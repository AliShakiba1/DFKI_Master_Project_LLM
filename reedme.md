# LLM Video Understanding & Activity Prediction Pipeline

This folder contains the core machine learning and natural language processing pipeline designed to process video recordings, perform zero-shot object detection, integrate verbal participant narrations, and leverage Large Language Models (LLMs) to predict actions and analyze behavior.

> 📁 **Future Multimodal Expansion:** This repository is structured to prioritize the `LLM` video pipeline. A separate `apple_watch` folder will be added to this project later to handle wearable sensor data integration.

---

## 📁 Repository Structure & Workflow

```
LLM/
├── 1. video_oo_chuck_0.ipynb     # Video processing, object detection & hand tracking
├── 2. narration.ipynb            # Narration alignment & text context propagation
├── 3. llm.ipynb                  # LLM reasoning engine & prediction compiler
├── 4. fd_silent_fix.json         # Intermediate dataset (aligned objects + narrations)
└── 5. output_predictions.json    # Final LLM output predictions mapped by chunk ID

```

## 🛠️ Step-by-Step File Documentation

### **1. Video to Object Detection (`video_oo_chuck_0.ipynb`)**

This notebook handles temporal downsampling, camera stream processing, and computer vision feature extraction.

* **Temporal Chunking:** Segments continuous video streams into uniform **3-second chunks**.


* **Direct Seeking:** Employs OpenCV hardware-level seeking to read only **1 representative frame** per chunk, bypassing intermediate frames to cut hardware I/O workloads by **98.6%**.


* **Zero-Shot Object Detection:** Runs **YOLOv8-World** configured with a custom vocabulary to detect specific task-related objects (e.g., scissors, measuring tape, pens).


* **Deduplication & Normalization:** Groups synonymous and varied detections into a standardized uppercase taxonomy (e.g., mapping "yellow tape measure" and "ruler" to `MEASURING_TAPE`).


* **Optional Gesture Tracking:** Leverages Google **MediaPipe Hand Landmarkers** to classify hand-action states (`REACH_OBJECT`, `GRASP_OBJECT`, `ACTIVE`, `IDLE`) utilizing multi-camera majority voting.



### **2. Narration Merging & Propagation (`narration.ipynb`)**

This notebook aligns the visual detection timeline with transcriptions of verbal participant narrations.

* **Metadata Fusion:** Matches transcribed textual narratives to their corresponding chronological video chunks.
* **Context Propagation:** Implements `propagate_narrations` to intelligently carry forward narration text into silent chunks. The narration text is only carried forward if there is object continuity (the same objects remain in view) or if an active hand gesture is detected.

### **3. Large Language Model Reasoning (`llm.ipynb`)**

The predictive brain of the pipeline. This script converts multimodal data into high-quality text prompts to classify human behavior.

* **Prototyping & Sampling:** Evaluates prompt configurations by running inference on **two random chunks** to review model behavior.
* **Batch Processor:** Implements production-ready loops to batch-process the entire video dataset through the LLM.

### **4. Integrated Dataset (`fd_silent_fix.json`)**

The physical structured output of the computer vision and narration pipelines. This intermediate JSON file serves as the clean feature representation fed into the LLM.

* **Data Schema Example:**
```json
{
    "chunk_id": 43,
    "time": "129-132",
    "cameras_data": {
        "gopro": { "processed_frame": 3975, "status": "success" }
    },
    "objects": ["LAPTOP", "SCISSORS", "KEYBOARD", "PEN"],
    "hand_motion": "GRASP_OBJECT",
    "participant_text": "[Propagated] Preparing to cut the packaging box"
}

```



### **5. Final Predicted Activities (`output_predictions.json`)**

The final compiled output dataset generated by the LLM. It links each `chunk_id` to a behavioral reasoning map and categorical prediction.

* **Data Schema Example:**
```json
{
    "chunk_id": 20,
    "time": "60-63",
    "predicted_activity": "Measuring Cardboard",
    "confidence": "High",
    "reasoning": "The participant is grasping a MEASURING_TAPE on a BOX, which directly supports their verbal statement of checking paper size."
}

```

## 🚀 Getting Started

### 1. Install Dependencies

Run the following commands to configure your local Python environment:

```bash
pip install opencv-python ultralytics mediapipe torch numpy<2 matplotlib --quiet

```

*Note: If utilizing the experimental hand-tracking module, download Google's pre-trained `hand_landmarker.task` file and place it in your working directory.*

### 2. Standard Execution Flow

To run the complete pipeline from scratch, execute the files in order:

1. Open and run **`1. video_oo_chuck_0.ipynb`** to generate temporal slices and extract detected objects.


2. Open and run **`2. narration.ipynb`** to merge, align, and propagate participant text transcripts.
3. Feed the resulting **`4. fd_silent_fix.json`** dataset into **`3. llm.ipynb`** to generate and export final behavior classifications into **`5. output_predictions.json`**.
